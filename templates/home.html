How it works

Follow these steps to secure your examination process

1. Generate RSA Key Pair SECURE

The Administrator initializes the system by generating strong 2048-bit RSA key pairs. The public key is used for encryption, while the private key remains secure for faculty decryption.

2. Encrypt Question Papers

Admins upload question papers which are instantly encrypted using the recipient's public key. Once encrypted, the raw content is never stored on disk.

3. Authorize Faculty Members

Access is governed by strict authorization. Admins must explicitly link faculty accounts to specific examination papers.

4. Decrypt & Access Paper

Authorized faculty use the secure private key to decrypt the paper in real-time. Every access is logged for audit transparency.
